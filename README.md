# HighcoreMiniShooter

Небольшой прототип 2D-шутера, сделанный в рамках тестового задания на позицию trainee Unity-разработчика. Автор: Ясев А.И.


КАК ИГРАТЬ

Управление:
WASD или стрелки — передвижение игрока.
Мышь — прицеливание, левая кнопка — стрельба.
Клавиши 1-9 и 0 — переключение оружия (порядок и количество слотов зависят от загруженного на игрока оружия).
R — перезарядка оружия вручную.

Цель: отстреливать врагов, следить за своим HP. Враги стоят на месте, получают случайное оружие из доступного пула и атакуют игрока раз в N секунд, не чаще, чем позволяет их оружие. При падении HP игрока до нуля появляется экран смерти с кнопкой рестарта — она перезапускает раунд без перезагрузки сцены.


ЧТО СДЕЛАНО ПО ТЗ

Игрок: HP, передвижение по плейну через WASD/стрелки, стрельба по клику мыши, переключение оружия по клавишам.

Четыре вида оружия: пистолет, дробовик, автомат, шокер. У каждого оружия настраиваемые через ScriptableObject параметры — урон, скорострельность, дальность, разброс, время перезарядки, размер обоймы, накладываемые эффекты.

Стрельба реализована снарядами (класс Projectile), а не хитсканом, снаряды выпускаются через пул объектов.

Враги: имеют HP, держатся на определённой дистанции от игрока (решил сделать их не статичными), получают случайное оружие из доступного пула (определён в конфиге спавнера для каждого типа противника), атакуют игрока раз в N секунд, не стреляют чаще, чем позволяет их оружие.

Архитектурные ожидания ТЗ соблюдены: данные оружия (WeaponData) отделены от логики оружия (IWeapon и реализации), HP и урон реализованы через IHealth/Health независимо от конкретных GameObject, ввод отделён от логики стрельбы через IInputProvider, MonoBehaviour используются только как Unity-обвязка (получение инпута, рендер, коллизии), а вся игровая логика — в обычных C# классах. Композиция зависимостей — через VContainer (DI-контейнер), а не через ручные ссылки и синглтоны.


ПАЙПЛАЙН ДОБАВЛЕНИЯ НОВОГО ОРУЖИЯ:

1. Создать и настроить SO конфиг оружия;
2. Если оружие накладывает статус эффект, добавить SO для него;
3. Назначить SO оружия игроку и\или желаемому типу врагов;
4. Если оружие должно стрелять по-особому, реализовать новый FirePattern (Scripts/Core/Weapons/FirePatterns); добавить новый тип в Enum WeaponTypes (Scripts/Data); добавить соответствующее создание паттерна оружия в метод CreatePattern классса WeaponFactory (Scripts/Core/ObjectManagement).


ДОПОЛНИТЕЛЬНЫЕ ФИЧИ, НЕ ВХОДИВШИЕ В ТЗ

HUD с отображением текущего HP, названия оружия, количества патронов, статуса перезарядки и счётчика убитых врагов, реализован по паттерну MVP (View/Presenter).

Экран смерти с рестартом без перезагрузки сцены: при смерти игра ставится на паузу через Time.timeScale, враги и активные снаряды очищаются, игрок и статистика сбрасываются, после чего игра возобновляется.

Динамическое назначение клавиш выбора оружия в зависимости от фактического количества оружия у игрока (от 1 до 10 слотов, клавиши 1-9 и 0), с предупреждением в лог, если оружия больше десяти.

Мировые UI-элементы (health bar врага, иконка статус-эффекта), которые следуют за персонажами, но не наследуют его поворот и отражение — вынесены в отдельный ответственный за это компонент.

Взвешенный рандом при спавне врагов и выдаче им оружия, а также нарастающая по времени и по числу убийств кривая интервала спавна.

Настройка CI/CD через GitHub Actions — можно играть прямо в браузере! (но не с мобильных устройств)
